HTML uses 32bit encoding, escaped with &# for decimal or &#x for hex

JavaScript has 3 escape options:

  • \x00 for 8 bit, must be 2 digits
  • \u0000 for 16 bit, must be 4 digits
  • \{000000} for 32 bit, may be 1 to 6 digits

Some glyphs are made up of conugate pairs for encoding to a lower standard. For example, the link character ðŸ”— (U+1F517) can be encoded with 128279 or 1F517 in 32 bit. For 16 bit it is made up of a conugate pair D83D and DD17 in hex, but cannot be encoded with decimal since the pair would be 55357 and 56599, which is more than 4 digits

Bits Dec Hex
32 128279 🔗 1F517 🔗
16 \u55357\u56599 \uD83D\uDD17
8 \xF0\x9F\x94\x97

Some symbols have standardized varition for alternative display. For example the Miscellaneous block uses variation selectors VS15 for text display and VS16 for emoji display. For example, the symbol for Aries uses base code U+2648 (dec 9800) and defaults display is an emoji - ♈ however selector VS15 (U+FE0E; 65038 dec) can be used for monochrome text styling ♈︎. Likewise the ☯ symbol U+262F default presentation is text however selector VS16 (U+FE0F, 65039 dec) can be used for emoji style with color ☯️
